    What is a URL?

    -uniform resource locater
    -reference to resource on the internet such as a file located on a machine connected to the network or database and command output.
    Two main parts:

    protocol identifier: example HTTP
    resource name: example.com is the resource

    resource name components:

    Host name- machine resource lives
    Filename- path to file on the machine
    Port number- port number to which to connect

      

    What does rails router do?

    -recognize URLs and dispatches to controllers action
    Example:
    -rails receives incoming request GET /patients/17
    -asks router to match it to controller action
    -if the first matching route is

    get '/patients/:id', to: 'patients#show'

    request is dispatched to the patients controller's show method with {id: '17' } in the params.

      

    What do active record validations ?

    -validate the state of objects before they go into the database
    -ensure only valid data is entered into the database

    *Model-level validation is the best way

    -database agnostic
    -cannot be bypassed by end user
    -convenient to test and maintain
    -rails makes it easy to use and provides helpers
    -can write your own

      

    Other ways to validate with active record?

    1)native database constraints: stored procedures that make the validation mechanisms database dependent and hard to test/maintain

    2)client side validation: generally unreliable if used alone

    3)controller-level validation: can become unwieldy and difficult to test and maintain

      

    Active record methods trigger validation

    Trigger validation and save only if valid:
    create, save, update

    skipping validations: lot's of these methods; be careful

    invalid? and valid? check just that

    errors[:attribute]: is attribute of an object valid? Used after validation and only inspects errors collection. Returns array.

      

    Validation Helpers do what?

    -single line of code that can add the same kind of validation to several attributes
    -used directly inside class definitions
    -active record pre defined

    examples:
    -acceptance (checkbox on user interface checked?)
    -validates_associated (associations with other models that need to be validated)
    -confirmation (email and password entered?)

    -exclusion, format, inclusion, length, numericality, presence, absence, uniqueness, validates-with......

    many more..look it up!

    Database basics

    schema: layout with tables and columns

    table: place to store multiple pieces of data sharing a common structure

    migration: change to to schema affecting one or more tables

    ORM(object relational mapper):fancy thing that talks to database. i.e. ActiveRecord

      

    Migrations ActiveRecord

    class Addplayer < ActiveRecord::Migration

    -has up down methods or a change method
    -use to make changes to database
    -must be numbered
    -class name matches file name after version

    003_add_player.rb

    Models ActiveRecord

    class User < ActiveRecord::Base
    belongs_to :game

    -might just contain the association
    -might also contain methods specific to the class like class Game has def win? method and belongs_to :player

    -need to require these models in your main app file which will contain majority of methods used to to affect or query the database based on what is defined or provided in the models

      

    What do associations do?

    -allow us to make connections between tables
    -rely on aspects of database schema, specifically the location of the foreign keys

    1-to1, 1-to-many, Many-to-many

      

    What are joins?

    a type of query involving data from more than one table

    What is the web?

    just a series of Requests and Responses

      

    HTTP request?

    includes:

    -HTTP verb (GET,PUT/PATCH,POST,DELETE)
    -domain name (computer used to access, server hosting website, or the website itself)
    -a path (to the file)
    -any needed parameters

      

    HTTP response?

    -very least, headers w/status code
    -usually some JSON data

      

    What do all HTTP requests and responses have?

    headers and a body

    -interested in headers of response

    -headers in request mostly used for authentication

      

    Headers?

    -status code
    -hostname
    -Interesting meta data?
    -server decides what else to include

      

    Body?

    -main content of request or response

    -GET(retrieve) request normally no body
    -POST(new) request normally have one

    **all RESPONSES other than for deletion normally have a body.

      

    GET ?

    HTTP verb

    -retrieve resource / read from CRUD

      

    PUT / PATCH ?

    HTTP verb

    -change a resource / update from CRUD

      

    POST ?

    HTTP verb

    -make a new resource / create from CRUD

      

    DELETE ?

    HTTP verb

    -delete resource / delete from CRUD

      

    What is HTTParty used for?

    to make HTTP requests with Ruby

    -includes class methods for sending request:
    get post put delete

      

    What do all request methods with HTTParty take?

    -optional parameters passed as key value pairs(hash)

    headers: specify hash of request headers

    body: hash of params to be sent in body of request (postdata)

    query: hash of params to use as a querystring
