    alias

    to alias a method or variable name in Ruby is to create a second name for the method or variable. Aliasing an be used either to provide more expressive options to the programmer using a class, or to help overrid methods and change behavior of the class or object. Ruby provides this functionality with the alias and alias_method keywords . alias :start :on reads as creates and alias for the on method called start. alias :old_on :on creates an alias for the on method called old_on

    and

    flow control operator with low precedence used in while loops and if statements. EX do x and y. If x and y. Not to be confused with && which is a boolean operator with high precedence.

    BEGIN

    break

    lets you alter the normal flow through a loop or iterator. break terminates the immediately enclosing loop; control resumes at the statement following the block. Terminate execution of a code block

    case

    case expression is a powerful beast. A multiway of if statements on steroids. As with if, case returns the value of the last expression executed, and you also need a then keyword if the expression is on the same line as the condition

    hashes

    Hashes are essentially arrays that aren't limited to integer indices. As a result, hash indices or keys can almost be any object. Hashes are indicated with curly braces containing key-value pairs. Although hashes generally resemble arrays, one important difference is that they don't generally keep their elements in a specific order. ex user["first_name"] = "Alex" user = { :name => "Alex Hawkins, :email => "alexhawkins@gmail.com", :password => "gata9ai" }
