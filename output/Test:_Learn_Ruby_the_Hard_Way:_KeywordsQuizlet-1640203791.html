38 Matching questions

  1. case

    INCORRECT: No answer given

    ANSWER: ai starts a case statement; this block of code will output a result and end when it's terms are fulfilled, which are defined with when or else.

  2. true

    INCORRECT: No answer given

    ANSWER: c The sole instance of the special class TrueClass. true encapsulates Boolean truth; however, <emph>all</emph> objects in Ruby are true in the Boolean sense (informally, they cause an if test to succeed), with the exceptions of false and nil.

  3. and

    INCORRECT: No answer given

    ANSWER: t A command that appends two or more objects together.

  4. break

    INCORRECT: No answer given

    ANSWER: k Gives an unconditional termination to a code block, and is usually placed with an argument.

  5. not

    INCORRECT: No answer given

    ANSWER: p Boolean negation. i.e. not true # false, not 10 # false, not false # true.

  6. next

    INCORRECT: No answer given

    ANSWER: b Bumps an iterator, or a while or until block, to the next iteration, unconditionally and without executing whatever may remain of the block.

  7. return

    INCORRECT: No answer given

    ANSWER: aa Inside a method definition, executes the ensure clause, if present, and then returns control to the context of the method call. Takes an optional argument (defaulting to nil), which serves as the return value of the method. Multiple values in argument position will be returned in an array.

  8. while

    INCORRECT: No answer given

    ANSWER: z Takes a condition argument, and executes the code that follows (up to a matching end delimiter) while the condition is true.

  9. undef

    INCORRECT: No answer given

    ANSWER: h Undefines a given method, for the class or module in which it's called. If the method is defined higher up in the lookup path (such as by a superclass), it can still be called by instances classes higher up.

  10. class

    INCORRECT: No answer given

    ANSWER: af Opens a class definition block, which can later be reopened and added to with variables and even functions.

  11. unless

    INCORRECT: No answer given

    ANSWER: ak The negative equivalent of if. i.e. unless y.score > 10 puts "Sorry; you needed 10 points to win." end.

  12. then

    INCORRECT: No answer given

    ANSWER: o Optional component of conditional statements (if, unless, when). Never mandatory, but allows for one-line conditionals without semi-colons.

  13. end

    INCORRECT: No answer given

    ANSWER: a Marks the end of a while, until, begin, if, def, class, or other keyword-based, block-based construct.

  14. retry

    INCORRECT: No answer given

    ANSWER: x Inside a rescue clause, causes Ruby to return to the top of the enclosing code (the begin keyword, or top of method or block) and try executing the code again.

  15. yield

    INCORRECT: No answer given

    ANSWER: l Called from inside a method body, yields control to the code block (if any) supplied as part of the method call. If no code block has been supplied, calling yield raises an exception.

  16. BEGIN

    INCORRECT: No answer given

    ANSWER: r Designates code that must be run unconditionally at the beginning of the program before any other.

  17. redo

    INCORRECT: No answer given

    ANSWER: ah Causes unconditional re-execution of a code block, with the same parameter bindings as the current execution.

  18. when

    INCORRECT: No answer given

    ANSWER: j Same as case.

  19. or

    INCORRECT: No answer given

    ANSWER: v Boolean or. Differs from || in that or has lower precedence.

  20. nil

    INCORRECT: No answer given

    ANSWER: w A special "non-object"; it is, in fact, an object (the sole instance of NilClass), but connotes absence and indeterminacy. nil and false are the only two objects in Ruby that have Boolean falsehood (informally, that cause an if condition to fail).

  21. elsif

    INCORRECT: No answer given

    ANSWER: ac Much like else, but has a higher precedence, and is usually paired with terms.

  22. super

    INCORRECT: No answer given

    ANSWER: ab Called from a method, searches along the method lookup path (the classes and modules available to the current object) for the next method of the same name as the one being executed. Such method, if present, may be defined in the superclass of the object's class, but may also be defined in the superclass's superclass or any class on the upward path, as well as any module mixed in to any of those classes.

  23. false

    INCORRECT: No answer given

    ANSWER: ae denotes a special object, the sole instance of FalseClass. false and nil are the only objects that evaluate to Boolean falsehood in Ruby (informally, that cause an if condition to fail.)

  24. else

    INCORRECT: No answer given

    ANSWER: y Gives an "otherwise" within a function, if-statement, or for-loop, i.e. if cats = cute, puts "Yay!" else puts "Oh, a cat."

  25. rescue

    INCORRECT: No answer given

    ANSWER: u Designates an exception-handling clause that can occur either inside a begin<code>/<code>end block, inside a method definition (which implies begin), or in modifier position (at the end of a statement).

  26. for

    INCORRECT: No answer given

    ANSWER: ad A loop constructor; used in for-loops.

  27. END

    INCORRECT: No answer given

    ANSWER: s Designates, via code block, code to be executed just prior to program termination.

  28. def

    INCORRECT: No answer given

    ANSWER: d Used to define a function.

  29. ensure

    INCORRECT: No answer given

    ANSWER: e Marks the final, optional clause of a begin/end block, generally in cases where the block also contains a rescue clause. The code in this term's clause is guaranteed to be executed, whether control flows to a rescue block or not.

  30. in

    INCORRECT: No answer given

    ANSWER: f Used with for, helps define a for-loop.

  31. alias

    INCORRECT: No answer given

    ANSWER: m To create a second name for the variable or method.

  32. do

    INCORRECT: No answer given

    ANSWER: g Paired with end, this can delimit a code block, much like curly braces; however, curly braces retain higher precedence.

  33. until

    INCORRECT: No answer given

    ANSWER: q The inverse of while: executes code until a given condition is true, i.e., while it is not true. The semantics are the same as those of while.

  34. self

    INCORRECT: No answer given

    ANSWER: aj The "current object" and the default receiver of messages (method calls) for which no explicit receiver is specified. Which object plays the role of self depends on the context.

  35. if

    INCORRECT: No answer given

    ANSWER: i Ruby's basic conditional statement constructor.

  36. begin

    INCORRECT: No answer given

    ANSWER: al Delimits a "begin" block of code, which can allow the use of while and until in modifier position with multi-line statements.

  37. module

    INCORRECT: No answer given

    ANSWER: n Opens a library, or module, within a Ruby Stream.

  38. defined?

    INCORRECT: No answer given

    ANSWER: ag A boolean logic function that asks whether or not a targeted expression refers to anything recognizable in Ruby; i.e. literal object, local variable that has been initialized, method name visible from the current scope, etc.
