    Length Method

    Ruby will return the length of the string (that is, the number of characters—letters, numbers, spaces, and symbols).
    For example: " I love espresso". length => 15

    Reverse Method

    Reversing input can be useful if you want to sort a list of values from highest to lowest instead of lowest to highest.
    For example : my_array = [3, 5 , 7 ,10]
    my_array.reverse => [10,7, 5, 3 ]

    Upcase & Downcase Method

    The .upcase and .downcase methods convert a string to ALL UPPER CASE or all lower case, respectively.
    For example: puts "eric".upcase => ERIC

    Single Line Comment

    A comment is a bit of text that Ruby won't try to run as code: it's just for humans to read. -
    # I am glad

    Multi - Line Comments

    You can write a comment that spans multiple lines by starting each line with a #, but there's an easier way. If you start with =begin and end with =end, everything between those two expressions will be a comment. Take a look at this example:
    =begin
    I' m happy
    I'm sad
    I am bored
    =end

    Gets Method

    gets. is the Ruby method that gets input from the user. When getting input, Ruby automatically adds a blank line (or newline) after each bit of input; chomp removes that extra line. For example: variable_name = gets.chomp

    String Interpolation

    It is essential to be able to replace placeholders within a string with values they represent. In the programming paradigm, this is called "string interpolation". In Ruby, string interpolation is extremely easy.
    For example:
    a = 7
    b = 8
    puts "The number # {a} is less than # {b}"

    If statement

    Ruby's if statement takes an expression, which is just a fancy word for something that has a value (like 4, true, or pants). If that expression is true, Ruby executes the block of code that follows the if. If it's not true (that is, false), Ruby doesn't execute that block of code: it skips it and goes on to the next thing.
    For example : if 1< 3
    print " I Love math"
    end

    •
      

    Else Statement

    The partner to the if statement is the else statement. An if/else statement says to Ruby: "If this expression is true, run this code block; otherwise, run the code after the else statement." Here's an example:
    if 1 > 2
    print "I won't get printed because one is less than two."
    else
    print "That means I'll get printed!"
    end

    •
      

    Elsif Statement

    The elsif statement can add any number of alternatives to an if/else statement.
    Example: if x < y # Assumes x and y are defined
    puts "x is less than y!"
    elsif x > y
    puts "x is greater than y!"
    else
    puts "x equals y!"
    end

    Unless Statement

    Sometimes you want to use control flow to check if something is false, rather than if it's true. You could reverse your if/else, but Ruby will do you one better: it will let you use an unless statement.
    For example:
    unless hungry
    # Write some sweet programs
    else
    # Have some noms
    end

    Comparison Operators

    In Ruby, we assign values to variables using =, the assignment operator. But if we've already used = for assignment, how do we check to see if two things are equal? Well, we use ==, which is a comparator (also called a relational operator). == means "is equal to." You can also check to see if two values are not equal using the != comparator.
    7 == 7
    7 != 8

    Less than: <
    Less than or equal to: <=
    Greater than: >
    Greater than or equal to: >=

    Logical Operators

    Logical operators are used to compare to boolean values.

    And - && :returns true if both boolean1 and boolean2 are true
    Or - ||
    Not - ! : returns the opposite of boolean

    Hashes

    Hashes are collections of key-value pairs. Like arrays, they have values associated with indices, but in the case of hashes, the indices are called "keys." Keys can be anything that's hashable, such as integers, strings, or symbols, but they must be unique for the hash they belong. The values to which keys refer can be any Ruby object.
    Syntax:
    hash = {
    key1 => value1,
    key2 => value2,
    key3 => value3
    }

    Global Substitution Method

    When we find "s", we want Ruby to replace every instance of "s" it finds with "th". We can do this with the .gsub! method, which stands for global substitution.

    Loops

    Ruby includes a while loop that will execute a block of code as long as its condition is true. When the condition becomes false, the code after the end of the loop will be executed.

    Assignment Operators

    You already know one assignment operator: =, which sets a variable. You can update a variable with additional assignment operators, however, which include +=, -=, *=, and /=.
    For example : counter += 1 => 1, 2 ,3 ,4 ,5.. etc

    1..15 - go up to 15 but include 15.
    1...10 means go up to but don't include 10.

    Until Loops

    Ruby includes an until loop that will execute a block of code as long as its condition is false. When the condition becomes true, the code after the end of the loop will be executed.
    Syntax
    until condition_is_false
    # do something
    end

    Next Keyword

    The next keyword can be used to skip over certain steps in the loop. For example:
    i = 20
    loop do
    i -= 1
    next if i % 2 == 1
    print "#{i}"
    break if i <= 0
    end

    The Iterator Method

    The loop iterator is the simplest, but also one of the least powerful. A more useful iterator is the .each method, which can apply an expression to each element of an object, one at a time.
    Syntax:
    object.each { |item| # Do something }
    object.each do |item| # Do something end

      

    For Loops

    The for loop is used to iterate an object. The Ruby .each method is preferred over the for loop because the for loop does not create a new scope for the object whereas the .each method does. The for loop is rare in Ruby.
    Syntax:
    for iterator in iterable_object
    # do something
    end

    Times Method

    The .times method is like a super compact for loop: it can perform a task on each item in an object a specified number of times.
    Example: 3.times {print"I love ruby"} => I love ruby , I love ruby , I love ruby

    Arrays

    An array is a Ruby data type that holds an ordered collection of values, which can be any type of object including other arrays.

    Example: array = [ 1, 2 , 3 ,4 ]

    Array Index

    Here's something interesting about arrays: each element in the array has what's called an index. The first element gets index 0, the next gets index 1, the one after that gets index 2, and so on. We can access elements of the array directly through these numbers.
    Syntax :
    array = [ 2, 3, 38 , 1]
    array [2] => 38

    Add to Hashes

    We can add to a hash two ways: if we created it using literal notation, we can simply add a new key-value pair directly between the curly braces. If we used Hash.new, we can add to the hash using bracket notation:
    pets = Hash.new
    pets.[key] = value

    pets = {
    "Stevie" => "cat",
    "Bowser" => "hamster",
    "Kevin Sorbo" => "fish"
    }

    puts pets["Stevie"] => "cat"

    variable = object.split

    Sort Method

    A Ruby array's sort method is pretty straightforward. If called without a block, it uses each objects' comparison operator (<=>) to return a new array with all the elements sorted. If called with a block, you may specify your own comparator as shown below
    Syntax:
    [1, 4, 3, 7, 2, 9].sort # => [1, 2, 3, 4, 7, 9]
    [1, 4, 3, 7, 2, 9].sort {|a, b| b <=> a} # => [9, 7, 4, 3, 2, 1]

    Appends the elements of other_ary to self.

    [ "a", "b" ].concat( ["c", "d"] ) #=> [ "a", "b", "c", "d" ]
    a = [ 1, 2, 3 ]

    Function / Method

    A method is a reusable section of code written to perform a specific task in a program. Methods are defined using the keyword def (short for "define").
    Syntax:
    def # method_name - header
    # code block that describes the method carried out - body
    end - # stop the code

    Calling a Method

    Defining a method is great, but it's not much use to you unless you call it.Ruby will start looking for the method with that name and try to execute the code inside it. (If Ruby doesn't find a method called # placeholder, it will return a NameError.
    Syntax:
    def method_name
    # do something
    end
    # call the method
    puts method_name

      

    Sort_by Method

    The sort_by method is slightly different. Its block only takes one parameter instead of two. Your array is then sorted by the results of each of the block calls.
    For example:
    [1, 4, 3, 7, 2, 9].sort_by {|i| -i} # => [9, 7, 4, 3, 2, 1]
    %w{ abcd ab abc abcde }.sort_by {|i| i.length} # => ["ab", "abc", "abcd", "abcde"]

    Converting to a Symbol

    A Ruby symbol is a thing that has both a number (integer) representation and a string representation. In general, to_sym returns the Symbol corresponding to an object. As sym is already a symbol, self is returned in this case. They are generated using the :name and :"string" literals syntax, and by the various to_sym methods.
    Syntax:
    1.9.2p320 :240 > "I am Ruby developer".to_sym
    => :"I am Ruby developer"
    1.9.2p320 :241 > "I am Ruby developer".intern
    => :"I am Ruby developer"

      

    Block

    A block is a chunk of code that lives inside a control statement, loop, method definition, or method call. It returns the value of its last line. In Ruby, blocks can be created two ways: with braces or with a do/end statement.

    Syntax:
    # Blocks that span only one line usually use the braces form
    objs.method { |obj| do_something }

    # Blocks that span multiple lines usually use the do/end form
    objs.method do |obj|
    # do first line
    # do second line
    # ...
    # do nth line
    end

    Each Method

    .each is a built in iterator function in Ruby. It loops through each item in a list, hash, or other iterable object allowing you to perform operations on that value. The block of an .each statement creates a new scope for your variable so you don't accidentally modify the original value.

      

    Collect Method

    The collect method takes a block and applies the expression in the block to every element in an array.
    Example : my_nums = [1, 2, 3]
    my_nums.collect { |num| num ** 2 }
    # ==> [1, 4, 9]

    Arguments and Parameters

    The argument is the piece of code you actually put between the method's parentheses when you call it,and the parameter is the name you put between the method's parentheses when you define it. For instance, when we defined square , we gave it the parameter n (for "number"), but passed it the argument 12 when we called it.
    Example: def square(n)
    puts n ** 2
    end
    square(12)
    # ==> prints "144"

      

    Splat Arguments

    Splat arguments are arguments preceded by a *, which signals to Ruby: "Hey Ruby, I don't know how many arguments there are about to be, but it could be more than one."
    Example: def what_up(greeting, * bros)
    bros.each { |bro| puts "#{greeting}, #{bro}!" }
    end

    what_up("What up", "Justin", "Ben", "Kevin Sorbo")
    #=> What up, Justin!
    What up, Ben!
    What up, Kevin Sorbo!
    ["Justin", "Ben", "Kevin Sorbo"]

      

    Return Statement

    Produces a output in the console.
    Example: def add(x = 9 ,y = 7)
    return x+y
    end
    add(7)
    add(9)
    #=> "16"

      

    The Combined Comparison Operator

    The combined comparison operator compares two Ruby objects. The combined comparison operator looks like this: <=>. It returns 0 if the first operand (item to be compared) equals the second, 1 if first operand is greater than the second, and -1 if the first operand is less than the second.

    A block that is passed into the sort method must return either 1, 0, -1. It should return -1 if the first block parameter should come before the second, 1 if vice versa, and 0 if they are of equal weight, meaning one does not come before the other (i.e. if two values are equal).

    Default Parameters

    (rev = false ) tells Ruby that the method has a second parameter, rev (for "reverse") that will default to false if the user doesn't type in two arguments.

    Nil

    Nil is one of two non-true values in Ruby. (Every other object is regarded as "truthy," meaning that if you were to type if 2 or if "bacon", the code in that if statement would be run.)

    It's important to realize that false and nil are not the same thing: false means "not true," while nil is Ruby's way of saying "nothing at all."

      

    Object id method

    The .object_id method gets the ID of an object—it's how Ruby knows whether two objects are the exact same object.

    Symbols and Hashes

    Symbols make good hash keys for a few reasons:

    They're immutable, meaning they can't be changed once they're created;
    Only one copy of any symbol exists at a given time, so they save memory;
    Symbol-as-keys are faster than strings-as-keys because of the above two reasons.
    Example:
    sounds = {
    :cat => "meow",
    :dog => "woof",
    :computer => 10010110,
    }

    •
      

    Hashes Methods

    Ruby includes two hash methods, .each_key and .each_value, that do exactly what you'd expect:

    my_hash = { one: 1, two: 2, three: 3 }

    my_hash.each_key { |k| print k, " " }
    # ==> one two three

    my_hash.each_value { |v| print v, " " }
    # ==> 1 2 3

    Case Statement

    The case statement, which will decide what actions to take based on what the user types in.
    Example:
    case language
    when "JS"
    puts "Websites!"
    when "Python"
    puts "Science!"
    when "Ruby"
    puts "Web apps!"
    else
    puts "I don't know!"
    end
